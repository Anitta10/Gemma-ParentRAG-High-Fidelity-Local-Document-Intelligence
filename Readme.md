This documentation explains how your "Smart Ingestion" and "Parent-Child" retrieval logic works.Ollama-Powered RAG with Parent Document RetrievalAn advanced RAG (Retrieval-Augmented Generation) system built with LangChain, Gemma 3, and Qwen 3 Embeddings. This project uses a Parent Document Retriever strategy to provide the LLM with better context while maintaining high-speed vector searches.🌟 Key FeaturesParent-Child Splitting: Uses RecursiveCharacterTextSplitter to create small 400-character child chunks for precise vector search, but retrieves larger 2000-character parent documents for the LLM to provide superior context.Smart Ingestion: Tracks the last processed PDF. If you swap a file in the data folder, the system automatically wipes the old vector store and indexes the new document.Source Attribution: Calculates the Euclidean distance (match score) between your query and the PDF. If the distance is above a threshold, it labels the response as "AI Generated" to prevent hallucination confusion.Persistent Storage:Vector Store: ChromaDB (Disk-based).Docstore: Pickle-based storage for parent documents.🛠️ PrerequisitesOllama Installed: Ensure Ollama is running on your machine.Models Downloaded:Bashollama pull gemma3:1b
ollama pull qwen3-embedding:0.6b
🚀 Getting StartedClone the repository and navigate to the project folder.Install dependencies:Bashpip install -r requirements.txt
Setup Environment:Create a .env file in the root directory:Code snippetDATA_PATH=./data
Add Data: Place your PDF file inside the ./data folder.Run the application:Bashpython main.py
🧠 How it WorksThe Retrieval StrategyInstead of just finding one small piece of text, the system:Searches for the most relevant Child Chunk.Identifies the Parent Document ID linked to that chunk.Sends the full Parent Document to the LLM (Gemma 3).Source Labeling LogicThe bot evaluates how "close" the question is to your PDF data:Match Score $\le$ 0.65: The bot attributes the answer to your PDF source.Match Score > 0.65: The bot warns you that it is relying on its internal AI training rather than the document.
